# 架构说明

这份公开版建议在论坛里只讲系统层次，不讲真实策略和私有接入细节。

## 推荐公开表达

```text
多源输入 -> 统一接口层 -> 网站展示层 -> 模拟盘执行层
```

## 各层职责

### 1. 多源输入

作用：
- 接入多路市场数据或信号来源
- 统一转成标准结构
- 用于交叉验证和运行监控

公开时建议只说：
- 主输入源
- 补充输入源
- 备用输入源

不要公开：
- 真实接口地址
- 真实限流方式
- Token 管理细节

### 2. 统一接口层

作用：
- 收敛不同来源的数据格式
- 给前端和执行层提供统一读取方式
- 降低后续替换数据源的成本

### 3. 网站展示层

作用：
- 状态页
- 概览页
- 日志和监控入口
- 论坛展示和产品演示

### 4. 模拟盘执行层

作用：
- 消费统一后的信号
- 做时效过滤、去重和约束控制
- 形成股票与期货的模拟盘自动化链路

公开时建议只讲：
- 有模拟盘验证
- 有执行约束
- 有状态回写

不要公开：
- 具体风控阈值
- 具体策略参数
- 具体下单判定细节

## 一句话版本

如果你在论坛里只想放一句话介绍架构，可以直接用：

`这套系统的重点不是单个策略脚本，而是把多源输入、统一接口、网站展示和股票/期货模拟盘自动化串成一条完整链路。`
