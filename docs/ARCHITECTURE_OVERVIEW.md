# 股票、期货实时数据获取架构说明

这份公开版文档重点说明的是实时数据获取与系统分层，不展开私有策略、内部参数或生产环境配置。

## 推荐公开表达

```text
实时数据源 -> 数据适配与标准化 -> 网站展示与状态监控 -> 模拟盘/自动化执行层
```

## 一、实时数据获取层

### 1. AllTick

定位：
- 适合作为主实时数据源
- 更偏向低延时、标准化和可程序化接入

适用场景：
- 股票实时行情接入
- 期货或商品类实时数据接入
- WebSocket 推送、REST 查询和历史数据补充

接入特点：
- 接口形式清晰，适合做统一数据接入层
- 便于直接进入网站展示、信号处理和模拟盘自动化链路
- 适合作为项目中的主实时行情源

官方入口：
- 官网：https://alltick.co/zh-CN
- 文档：https://en.apis.alltick.co/

### 2. 东方财富 EMT API / 机构交易平台

定位：
- 适合作为国内市场的一体化量化接入平台
- 可同时服务于研究、仿真、实盘和数据接入场景

适用场景：
- 股票实时行情与历史数据获取
- 量化终端接入
- 策略研究、仿真和交易联动

接入特点：
- 更适合国内证券与期货量化终端生态
- 平台能力完整，适合需要终端、SDK、回测和仿真协同的场景
- 适合作为主数据源之外的增强型接入方案

官方入口：
- 申请入口：https://emt.18.cn/business/business-apply-all
- 帮助文档：https://emt.18.cn/api/quant-help/guide/guide.html

### 3. 通达信行情体系

定位：
- 适合作为国内证券行情补充源或备用接入层
- 更适合 A 股分时、K 线和终端行情体系的辅助接入

适用场景：
- A 股行情补充
- 备用扫描链路
- 与券商终端或本地行情工具协同使用

接入特点：
- 在国内证券行情软件生态中覆盖面广
- 适合作为补充源、校验源或备用链路
- 工程上可结合终端行情服务或协议兼容客户端接入

官方入口：
- 官网：https://www.tdx.com.cn
- 公司介绍：https://www.tdx.com.cn/article/about.html

## 二、建议的数据分层方式

### 1. 实时数据接入层

职责：
- 接入股票与期货实时行情
- 对接不同来源的数据接口
- 保持主源、补充源和备用源的分工清晰

### 2. 数据适配与标准化层

职责：
- 统一不同来源的字段结构
- 统一时间、代码、价格和成交量等核心格式
- 为上层网站、日志、模拟盘和自动化流程提供一致的数据协议

### 3. 网站展示与状态监控层

职责：
- 展示实时数据状态
- 展示数据来源运行情况
- 展示信号快照、日志和系统状态

这一层的价值不只是展示，而是为日常排障、运行监控和链路验证提供可视化入口。

### 4. 模拟盘与自动化执行层

职责：
- 消费标准化后的实时数据或信号
- 执行去重、过滤、约束和状态回写
- 形成股票与期货的模拟盘自动化闭环

公开说明时建议强调：
- 有完整链路验证
- 有模拟盘或仿真环境验证
- 有网站监控和状态回写

不建议公开：
- 具体策略参数
- 私有风控阈值
- 内部接口地址或鉴权细节

## 三、一句话版本

这套系统的重点，不是单一的数据接口或单一策略，而是把股票、期货实时数据获取、标准化适配、网站监控展示和模拟盘自动化串成一条稳定的工程链路。
