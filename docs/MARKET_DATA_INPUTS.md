# 多源数据输入说明

这份说明只介绍公开可分享的接入定位、功能分工、接入要求和工程取舍，不包含私有配置、真实 Token 或策略实现细节。

## 1. AllTick

### 在系统中的定位

主实时数据源，优先承担实时推送、主扫描和自动化链路输入。

### 主要能力

- 提供标准化实时行情接入
- 支持 WebSocket 和 REST
- 支持历史数据补充
- 适合作为主实时数据源进入网站、信号处理和模拟盘自动化链路

### 接入要求

- 需要注册账号并获取 API Key / Token
- 需要根据调用频率、数据品类和连接数量选择套餐
- 需要根据业务规模评估订阅数量、限流和连接限制

### 工程使用感受

AllTick 更适合作为“主实时源”，因为它的接口形态更标准、接入路径更清晰，也更适合作为后端服务和自动化系统的直接输入。

## 2. 东方财富 EMT API / 机构交易平台

### 在系统中的定位

国内市场的一体化平台型接入方案，可作为增强型数据源和业务协同平台。

### 主要能力

- 提供股票实时行情与历史数据能力
- 支持终端、SDK、仿真和交易协同场景
- 更适合研究、仿真、终端联动和平台化接入

### 接入要求

- 需要通过 EMT 官方入口申请业务
- 需要根据平台要求完成账号、权限和终端环境配置
- 需要结合官方文档接入对应平台能力

### 工程使用感受

东方财富 EMT API 更像一套完整业务平台，而不是单一行情接口。它更适合需要国内量化终端、仿真和研究协同的团队场景。

## 3. 通达信 TdxQuant / tqcenter 官方公开接口

### 在系统中的定位

国内证券行情补充源、备用源和终端型接入层。

### 主要能力

- 通过 TdxQuant / `tqcenter` 官方 Python API 获取数据
- 支持股票行情、财务和市场交易数据
- 支持期货行情及历史产品数据
- 适合作为补充源、备用扫描链路和数据校验链路

### 接入要求

- 需要安装支持 TQ 策略功能的通达信金融终端或专业研究版
- 需要按官方文档配置运行环境并调用 `tqcenter` 接口
- 需要根据终端环境处理本地运行和接口调用约束

### 工程使用感受

通达信官方公开接口更适合作为终端型接入方案和补充源。它在国内 A 股生态中很实用，但从工程形态上看，更适合作为辅助链路，而不是唯一主实时源。

## 4. 三类数据源的差异总结

### 从接入形态看

- AllTick 更偏标准 API 形态
- 东方财富 EMT API 更偏平台型接入形态
- 通达信 TdxQuant / tqcenter 更偏终端型接口形态

### 从工程定位看

- AllTick 更适合作为主实时数据源
- 东方财富 EMT API 更适合作为平台化增强接入
- 通达信更适合作为补充源、备用源和校验源

### 从部署方式看

- AllTick 更适合直接对接后端服务
- 东方财富 EMT API 更适合与终端、仿真和业务流程联动
- 通达信更适合建立在终端环境之上的本地化接入

### 从系统分工看

如果只用一句话概括三路输入的分工，可以写成：

```text
AllTick 负责主实时输入，东方财富 EMT API 负责平台化增强接入，通达信官方公开接口负责补充、备用和校验链路。
```
