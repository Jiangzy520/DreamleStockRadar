# 公开版使用文档

这份目录是用于 GitHub 公开展示的本地公开版。

公开版保留了页面结构、入口、三源切换说明、截图导入、自选池编辑、桥接页占位和通知推送配置页，但已经移除了真实信号记录、真实账户信息、真实策略参数、真实日志和生产 API 凭据。

## 1. 页面入口

- 股票信号页：`http://127.0.0.1:18080/push`
- 期货信号页：`http://127.0.0.1:18081/`
- 股票桥接：`http://127.0.0.1:8792/`
- 期货桥接：`http://127.0.0.1:8792/futures`
- 推送配置：`http://127.0.0.1:8792/notifications`

## 2. 三源切换

股票信号页保留了三个来源版本的切换说明：

- AllTick
- 通达信
- 东方财富

当前公开版是“单页面结构 + 三源切换标签”的展示模式。
也就是说，页面会随着来源切换更新标题和说明，但不会显示真实生产记录。

## 3. 截图导入与自选池

股票信号页保留了以下功能：

- 批量拖入图片
- 从多张图片提取代码
- 读取当前自选
- 保存当前备份
- 写入明天池子并重新分配

当前逻辑说明：

1. 上传多张截图
2. OCR 自动提取 6 位股票代码
3. 去重、排序后写入自选池
4. 同步到 AllTick / 通达信 / 东方财富 三个版本

注意：

- 这不是三套互相独立的自选池，而是一套共用池子同步到三源。
- 如果你在公开版里实际操作过导入和保存，上传 GitHub 前请再次确认是否需要清空这些本地生成结果。

## 4. 推送配置

通知中心已经支持三种机器人：

- 飞书
- 钉钉
- 企业微信

使用方式：

1. 在对应平台创建群机器人
2. 复制 Webhook 地址
3. 打开 `推送配置` 页面
4. 填入 Webhook
5. 勾选要接收的事件类型
6. 点击“保存推送配置”
7. 在右侧“测试发送”里发送测试消息

支持的事件类型：

- `stock_signal`：股票信号
- `futures_signal`：期货信号
- `stock_trade_signal`：股票交易信号
- `futures_trade_signal`：期货交易信号

统一接收接口：

```http
POST http://127.0.0.1:8792/api/notifications/dispatch
```

## 5. 当前通知中心做到哪一步

现在已经完成：

- 通道配置保存
- 通道启停和事件勾选
- 手动测试消息发送
- 统一通知接口接收
- 飞书 / 钉钉 / 企业微信三通道路由

当前还没完成：

- 股票实时信号脚本自动接入通知中心
- 期货实时信号脚本自动接入通知中心
- 股票成交回报自动接入通知中心
- 期货成交回报自动接入通知中心

也就是说：

- 手动测试可以发
- 外部脚本手动 `POST` 到统一接口可以发
- 但“全站自动联动推送”还需要再做一次正式接入

## 6. 桥接页说明

股票桥接和期货桥接页面当前保留的是：

- 页面布局
- 模块入口
- 说明卡片
- 占位指标

公开版移除的是：

- 真实账户信息
- 真实持仓
- 真实盈亏
- 真实成交记录
- 真实策略参数
- 真实日志输出

期货信号页中的按钮在公开版里保留了入口和文案，但不会执行真实扫描、拉价或回测任务。

## 7. GitHub 上传前检查

上传前建议检查：

1. `push_xtp_bridge/state/notification_channels.local.json` 是否为空或是否需要删除
2. 自选池文件是否被你本地重新写入过
3. 是否留下了截图导入后的临时结果
4. 是否留下了新的 CSV、日志、运行状态文件
5. 页面中是否还出现真实 Webhook、真实账户、真实密码、真实服务器地址

`.gitignore` 已经忽略了一部分本地文件，但如果你是“手动打包整个目录”发给别人，而不是用 Git 提交，那么本地文件依然会跟着走。

## 8. 建议的下一步

如果你后面要把这套系统做成“客户可直接使用”的版本，建议按这个顺序继续：

1. 把股票信号自动接入通知中心
2. 把期货信号自动接入通知中心
3. 把股票交易回报自动接入通知中心
4. 把期货交易回报自动接入通知中心
5. 把本地 `127.0.0.1` 地址改成可配置地址
6. 把三源自选池做成“共用池”或“三套独立池”二选一的明确模式
