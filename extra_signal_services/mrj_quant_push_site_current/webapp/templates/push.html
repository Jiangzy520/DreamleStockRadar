站点使用说明
下面这份说明重点解释页面入口、三源切换、截图导入、自选池同步和推送配置的使用方式。

1. 页面入口怎么用

首页顶部的快捷入口用于在不同模块之间跳转，建议按下面顺序查看。

  • `股票信号页`：查看股票信号页主界面、三源切换、自选池编辑和完整信号表结构。
  • `期货信号页`：查看期货盯盘页的结构、指标卡片、表格区域和按钮入口。
  • `股票桥接` / `期货桥接`：查看桥接面板布局、卡片区域和导航入口。
  • `推送配置`：客户自己填写飞书、钉钉、企业微信 Webhook 的地方。

2. 三源切换怎么理解

页面支持 AllTick、通达信、东方财富 三个来源视角，可用于展示不同来源标签下的同一套页面结构。

  • 点击页面里的三源切换按钮，可以切换当前查看的来源视角。
  • 切换后，完整信号表标题、来源说明、页面文案会跟着切换。
  • 三源当前使用同一套页面结构，重点展示来源标签、入口关系和模块组织。
  • 如果后续需要分成独立来源版本，可再按来源拆分自选池和信号链路。

3. 截图导入与自选池同步

`从多张图片提取代码` 和 `写入明天池子并重新分配` 两步功能都保留着，便于演示导入流程。

  • 先把多张截图拖进去，或者手动选择图片文件。
  • 点击 `从多张图片提取代码` 后，系统会做 OCR，提取 6 位股票代码并自动去重。
  • 确认文本框里的代码没问题后，点击 `写入明天池子并重新分配`。
  • 当前逻辑是：写入一个共用自选池，并同步到 AllTick / 通达信 / 东方财富 三个版本。
  • 导入完成后，可以再切换三源标签查看同一套池子在不同来源说明下的展示方式。

4. 环境准备： 下载 VS Code 并安装 Codex 扩展

这部分对应上面的固定导航入口，说明推荐的查看和整理方式。

  • 建议用最新版 VS Code 打开项目目录，便于浏览 HTML、Python、systemd、nginx 等文件。
  • 安装 Codex 扩展后，可以更快定位模板、接口、服务脚本和页面入口之间的关系。
  • 如果只是查看演示页结构，不需要额外接入行情、桥接或通知服务。
  • 如需进一步接入数据、通知或桥接能力，可按部署环境补充对应配置。

5. 股票、期货实时数据获取说明

当前页面保留了股票三源支持和期货页面结构，用于说明多来源接入后的展示方式。

  • 股票页现在保留 AllTick、通达信、东方财富 三源切换能力与说明文案。
  • 期货页保留盯盘页、桥接页、指标卡和表格结构，便于展示站点的期货模块。
  • 如需启用实时数据，可按你的部署环境接入对应的数据源服务。
  • 当前这套页面更适合产品展示、功能演示和文档说明。

6. 股票、期货模拟盘账号与接口说明

这部分对应你强调要保留的第三条导航，用来说明账户、桥接和通知接口的接入位置。

  • 股票桥接、期货桥接、推送配置页面都保留入口和页面结构，便于说明模块关系。
  • 飞书、钉钉、企业微信的推送接口支持客户自行填写。
  • 模拟盘账号、交易接口和通知参数建议按实际部署环境单独配置。
  • 如需扩展自动化能力，可继续把股票信号、期货信号和交易回报统一接入通知中心。

7. 推送配置怎么接

`推送配置` 页面已经支持三种机器人：飞书、钉钉、企业微信，客户可以自己填写地址。

  • 先在各自平台创建群机器人，拿到 Webhook 地址。
  • 把地址填进通知中心，勾选需要接收的事件类型后点击 `保存推送配置`。
  • 可在通知中心用 `发送测试消息` 验证当前通道是否可达。
  • 通知中心支持四种事件：股票信号、期货信号、股票交易信号、期货交易信号。
  • 统一接收接口是 `POST /api/notifications/dispatch`，后端已经能接收并路由。
